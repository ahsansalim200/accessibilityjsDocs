"The power of the Web is in its universality.
Access by everyone regardless of disability is an essential aspect."
- Tim Berners-Lee, W3C Director and inventor of the World Wide Web

Accessibility can be defined as the ease with which people with disabilities can perceive, understand, navigate, interact with, and contribute to the web. The Web is fundamentally designed to work for all people, whatever their hardware, software, language, location, or ability. When the Web meets this goal, it is accessible to people with a diverse range of hearing, movement, sight, and cognitive ability.

Thus the impact of disability is radically changed on the Web because the Web removes barriers to communication and interaction that many people face in the physical world. However, when web sites, applications, technologies, or tools are badly designed, they can create barriers that exclude people from using the Web.

This JavaScript library is a step towards the goal of Web for all! You are entitled to contribute to this library and be a hero to the world.
This documentation is to help you achieve web accessibility. Please go through the documentation carefully to understand how this library can help you save thousands of hours of coding.

Focus Management #back to top

This module controls focus on the screen and flow management among different links, headings and input fields. Tab-index and keyboard commands to implement focus management have been programmed in this. This kind of control is imperative for Screen Readers which are used as a hearing device to blind or low vision people.

In case of pop ups and alert boxes, this module is used to control focus and escape action from them. Toggling action is customizable to enter key, mouse click or tab. There is also an option for timeout for actions.

Only four attributes are required to manage flow control using Accessibility JS:

  • data-accessibility-focus-next
    This attribute is used to tell where the focus should shift to next.
  • data-accessibility-focus-target
    This data attribute locates specific elements when other attributes point to it.
  • data-accessibility-focus-return
    Escape action from a pop up requires an element to be focused on. This data attribute points to this start element.
  • data-accessibility-toggle-method
    This attribute is used to customize toggling action event.

1
2
3
4
<a href="#" data-toggle="modal" data-target="#accFocusPopin" class="btn btn-primary" data-accessibility-focus-return="first" data-accessibility-toggle-method="click" data-accessibility-focus-next="textfield1">Popin</a>
 
 
<button type="button" class="btn btn-default" data-dismiss="modal" data-accessibility-focus-target="button2" data-accessibility-focus-next="button3" data-accessibility-toggle-method="click">Close</button>

Example

Button1 Button2 Popin Button3

The focus shifts to the pop in window when it is called. When escape action is called the focus returns to the element which was in focus before the event was triggered.

Menu#back to top

Accessible menus improve site’s usability and overall user experience. Up and down arrow keys should work for navigation. Enter key helps in item selection and escape key helps in exiting a menu. In Accessibility.js these things are achieved using only two attributes in the HTML page.

Only two attributes are required to make menus accessible:

  • data-accessibility-menu
    Used to define each menu as an individual entity.
  • role="menuitem"
    This attribute defines all the menu items to the JS library.

MenuBar #back to top

A menu bar is a graphical control element which contains drop-down menus. An accessible menu bar would consist of accessible menus which can be traversed through easily using only the left and right arrow keys. This does not require many extra tags or attributes on the HTML side making it very easy to be used by developers who are not comfortable with JavaScript.

Only two new attributes are required to make menubar accessible:

  • data-accessibility-menubar
    This attribute is used to call the function for menubar accessibility.
  • data-accessibility-menu-item-parent
    Used to define each menu individually in a menubars so that account of all the menus can be kept.

Virtual Tables(includes Seat Maps) #back to top

An aircraft seat map or seating chart, is a diagram of the seat layout inside a passenger airliner. They are often published by airlines for informational purposes, and are of use to passengers for selection of their seat at booking or check-in. These table-like structures do not conform to any particular design form and are independent of element construct.
The seats should be accessible by the keyboard regardless of their HTML design. Using the arrow keys, navigation in the seat map should be possible.

Three new attributes are required to have accessible seat maps:

  • data-accessibility-seat-map
    This attribute is used to call the function for seat map accessibility.
  • data-accessibility-row-index
    This helps in configuring the current position in the map row-wise.
  • data-accessibility-col-index
    This helps in configuring the current position in the map column-wise.

Color Contrast #back to top

Colour Contrast of the web page is a very crucial for increased usability. For a person with low vision, having a high colour contrast helps interact with the UI. For example, let's take guideline 1.4, which is about making it easier for users to see and hear content including separating foreground from background.

However, a developer may choose to keep only part of the page as a high colour contrast, therefore it is important to keep the CSS for colour contrast for different sections separate. Using Accessibility JS this can be achieved only by adding CSS classes and a switch between normal and accessible design is possible by pressing a key combination of CTRL + Q.

Only one attribute is required to include high colour contrast CSS classes:
data-accessibility-class
This attribute is used to call the function and the value of the attribute is used to add the CSS class.

1
2
3
<div class="colorContrastDemo2" data-accessibility-class="class2">
    <span> Color Contrast Demo2</span>
</div>

Skip Links #back to top

Providing links that allow the user to skip directly to the main content, bypassing the navigation, enhances the accessibility of your web site. This is recommended for blind or visually impaired users, people who use screen readers, and also for text-browsers, mobile phones and PDAs.

By using the key combination of CTRL + Y a pop up window comes up with the main headings of all the web page and an internal link to all these headings.

Two new attributes are required to have a Skip Link component:

  • data-accessibility-skipLink
    Preferably included in the body tag, this attribute is responsible for calling the function from JS library.
  • data-accessibility-skip-link
    This attribute makes up the list of the main headings and acts as links within the page.

1
<h2 class="dark-text" tabindex="0" data-accessibility-skip-link="Focus" name="Focus">Focus Management <a href="#top">#back to top</a><hr></h2>

Example

Press the key combination to see the pop up window. The window contains an internal link within the page to the main headings.
